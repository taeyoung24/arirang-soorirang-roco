### 1. 언어 및 문체
- **모든 내용은 반드시 한국어로 작성할 것.**
- **간결한 명사형 어미 사용:** 문장의 끝나는 형태를 명사형 어미(예: `~함`, `~할 것`, `~예정` 등)로 처리하여 간결하게 유지할 것.

### 2. 제목 (Title) 작성 규칙
- **카테고리 분류 (대괄호 등) 사용 금지:** 제목에 `[Feature]`, `[Bug]` 등과 같은 대괄호를 활용한 카테고리 태그를 붙이지 않고 본론만 깔끔하게 작성할 것.
- **간결하고 명확한 작성:** 
  - (예시) `이메일 인증 API 메일 발송 어뷰징 방지 (Rate Limit 적용)`
  - (예시) `API 엔드포인트 JSDoc 설명 및 주석 추가 작업`

### 3. 마크다운(Markdown) 포맷팅
마크다운 형식은 기본적으로 자유로우나 가독성을 위해 다음 구조를 준수할 것을 제안합니다.
- **문단 제목 (단락 구분):** H3 (`###`) 태그를 사용할 것.
- **하위 항목 나열:** 불릿 리스트(`-`)를 사용하여 간결하게 나열할 것.
- **항목 제목 강조:** 리스트 항목의 시작 부분에 소제목이 들어가는 경우 **Bold(`**`)** 처리하여 구분할 것.
  - (예시) `- **리스트 항목의 제목은:** 텍스트 내용`

### 4. 이슈 본문 필수 포함 요소
작업자가 이슈를 온전히 이해하고 즉시 작업에 착수할 수 있도록 다음의 요소들을 적절히 포함하여 작성할 것.
- **배경 및 개요:** 이 이슈가 왜 발행되었으며, 어떤 문제를 해결하고 무엇을 달성하고자 하는지에 대한 설명.
- **세부 사항:** 상세한 기획, 구체적인 작업 내용, 작업해야 할 위치(파일 경로 등), 그리고 주요 구현 사항.
- **작업 체크리스트:** 실질적인 구현 태스크가 뚜렷하다면 마크다운 체크리스트 문법(`- [ ]`)을 적절히 활용하여 나열할 것.
- **참고사항:** 작업 시 확인해야 할 링크, 관련 기록, 또는 문서 부분 등을 기재.
- **정보가 부족한 경우:** 필수 포함 요소 중 유저가 제공한 내용이 없거나 불충분한 항목이 있다면, 해당 항목을 비워두거나 유저에게 어떤 정보가 필요한지 **추가로 질문할 것**.

### 5. 아웃풋(Output) 저장 위치
- 완성된 이슈 내용은 **반드시** 프로젝트 내의 `.agents/outputs/` 디렉토리에 마크다운(`.md`) 형식의 파일로 생성하도록 에이전트 또는 시스템에 지시할 것.
